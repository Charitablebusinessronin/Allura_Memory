## Executive Summary

This specification maps every visual decision in the Amaysin Pro 5-page brand presentation template to the Allura Memory identity system. Rand will use this as the production blueprint for HTML/CSS assembly. All values are explicit: hex codes, px/rem values, font weights, and CSS rules. No ambiguity.

**Amaysin Pro baseline:** Bold, high-contrast, editorial, luxury-meets-startup aesthetic. Dark gradients, red/gold/teal accent system, Bebas Neue display type, dramatic shadow cards.  
**Allura target:** Warm, inviting, community-centric. Light-first, yellow/blue/green accent system, Outfit + Inter type pairing, 3-layer editorial shadow system.

The adaptation is not a 1:1 swap — it is a translation of *visual role* and *hierarchy function* from one brand language to another.

### 1.2 Gradient Backgrounds

Amaysin uses dramatic full-bleed gradients (red→gold, gold→teal, dark radial). Allura replaces these with warm, luminous gradients that suggest community warmth and digital light.

**Slide 1 — Hero Gradient (Brand Overview)**
```css
background: linear-gradient(
  135deg,
  #F5F5F5 0%,
  #FFF8E1 35%,
  #E6F4F1 70%,
  #F5F5F5 100%
);
```
- Warm Yellow (#FFC300) at 10% opacity creates the golden center glow
- Deep Blue (#0581A7) at 8% opacity creates the cool secondary shift
- Purpose: Inviting, warm, "digital hearth" feeling rather than aggressive luxury

**Slide 2 — Color System Background**
```css
background: linear-gradient(
  180deg,
  #F5F5F5 0%,
  #FFFFFF 50%,
  #F5F5F5 100%
);
```
- Clean, neutral stage for color swatches to read accurately
- No tinting that would skew swatch perception

**Slide 3 — Logo Chooser Background**
```css
background: radial-gradient(
  circle at 30% 50%,
  rgba(255, 195, 0, 0.06) 0%,
  transparent 60%
),
#F5F5F5;
```
- Subtle warm spotlight draws eye to primary logo placement
- No gradient on the grid itself — cards float on neutral ground

**Dark Mode Title Slide (Alternate)**
```css
background: linear-gradient(
  160deg,
  #142329 0%,
  #1A2E35 50%,
  #142329 100%
);
```
- Dark Gray base with a 5% luminosity lift in the center
- Used only for title/hook slides, never for content-dense pages

## 2. Typography Translation

### 2.1 Font Family Swap

| Amaysin | Allura | Role |
|---------|--------|------|
| Bebas Neue | Outfit | Display / Headings |
| Montserrat | Inter | Body / UI |

**Google Fonts CDN:**
```html
<link href="https://fonts.googleapis.com/css2?family=Outfit:wght@400;600;700&family=Inter:wght@400;500;600&display=swap" rel="stylesheet">
```

### 2.2 Size Mapping — Hero to Body

Amaysin's Bebas Neue 120px hero headlines achieve visual impact through extreme condensed height and uppercase letterforms. Outfit is a geometric sans with rounded terminals and a larger x-height. It feels friendlier but occupies more horizontal space at the same font-size. The mapping below preserves *visual weight* and *hierarchy scale*, not literal px values.

| Amaysin Usage | Amaysin Size | Allura Size | Allura Weight | Line Height | Letter Spacing | Notes |
|---------------|-------------|-------------|---------------|-------------|----------------|-------|
| Hero display | 120px / 7.5rem | 96px / 6rem | 700 | 1.05 | -0.03em | Outfit needs tighter tracking at huge sizes to feel intentional |
| Section title | 72px / 4.5rem | 56px / 3.5rem | 700 | 1.1 | -0.02em | H1 equivalent |
| Slide headline | 48px / 3rem | 40px / 2.5rem | 600 | 1.15 | -0.01em | H2 equivalent |
| Card title | 32px / 2rem | 28px / 1.75rem | 600 | 1.2 | 0 | H3 equivalent |
| Subheading | 24px / 1.5rem | 20px / 1.25rem | 600 | 1.3 | 0 | Small headings, labels |
| Body large | 20px / 1.25rem | 18px / 1.125rem | 400 | 1.6 | 0 | Lead paragraphs |
| Body | 16px / 1rem | 16px / 1rem | 400 | 1.6 | 0 | Standard body — unchanged, Inter is optimal here |
| Caption / Meta | 14px / 0.875rem | 14px / 0.875rem | 500 | 1.4 | 0.01em | Inter Medium for readability |
| Micro | 12px / 0.75rem | 12px / 0.75rem | 500 | 1.4 | 0.02em | Badges, timestamps |

### 2.3 Letter-Spacing & Line-Height Adjustments for Outfit

Outfit's geometric construction has wide counters and rounded terminals. At display sizes it can feel loose unless tracked tightly. At body sizes it needs room to breathe.

**CSS Custom Properties for Typography:**
```css
:root {
  --font-display: 'Outfit', sans-serif;
  --font-body: 'Inter', sans-serif;

  /* Display — tight, intentional */
  --display-hero: 700 6rem/1.05 var(--font-display);
  --display-section: 700 3.5rem/1.1 var(--font-display);
  --display-headline: 600 2.5rem/1.15 var(--font-display);

  /* Body — open, readable */
  --body-large: 400 1.125rem/1.6 var(--font-body);
  --body-standard: 400 1rem/1.6 var(--font-body);
  --body-caption: 500 0.875rem/1.4 var(--font-body);

  /* Tracking tokens */
  --tracking-tight: -0.03em;
  --tracking-display: -0.02em;
  --tracking-headline: -0.01em;
  --tracking-normal: 0;
  --tracking-wide: 0.01em;
}
```

**Specific Adjustments:**
- Hero text (`--display-hero`): `letter-spacing: -0.03em` is mandatory. Without it, Outfit at 96px feels childish rather than bold.
- Section titles (`--display-section`): `letter-spacing: -0.02em` maintains punch while allowing slightly longer titles.
- Never use Outfit below 20px — Inter is the body typeface for a reason.
- All headings: `text-transform: none` — Allura does not use all-caps. It feels colder and less approachable.

### 2.4 Uppercase → Sentence Case Rule

Amaysin uses `text-transform: uppercase` extensively (Bebas Neue is an all-caps font by design). Allura replaces this with sentence case:

| Amaysin Pattern | Allura Pattern |
|-----------------|----------------|
| "BRAND OVERVIEW" | "Brand Overview" |
| "COLOR SYSTEM" | "Color System" |
| "TYPOGRAPHY" | "Typography" |
| "01 / 05" slide numbers | "01 / 05" unchanged — numeric labels are neutral |

Exception: The 1-2-3-4-5 page navigation circles remain numbers only (no text transform needed).

## 3. Layout Adaptations

### 3.1 Hero Section (Slide 1 — Brand Overview)

**Amaysin baseline:** Full-bleed dark gradient, centered massive headline, red accent underline, floating 3D phone/mockup on right.

**Allura adaptation:**

```css
.hero-section {
  background: linear-gradient(135deg, #F5F5F5 0%, #FFF8E1 35%, #E6F4F1 70%, #F5F5F5 100%);
  min-height: 100vh;
  display: grid;
  grid-template-columns: 1fr 1fr;
  align-items: center;
  padding: 4rem 6rem;
  gap: 4rem;
}

.hero-headline {
  font: var(--display-hero);
  letter-spacing: var(--tracking-tight);
  color: #142329;
  max-width: 600px;
}

.hero-accent-bar {
  width: 80px;
  height: 4px;
  background: #FFC300;
  border-radius: 2px;
  margin: 1.5rem 0;
}

.hero-subhead {
  font: var(--body-large);
  color: #0581A7;
  max-width: 480px;
}

.hero-visual {
  /* Logo mark or community imagery */
  display: flex;
  justify-content: center;
  align-items: center;
}

.hero-logo-mark {
  width: 400px;
  max-width: 100%;
}
```

**Key changes from Amaysin:**
- Background: Dark radial → warm multi-stop gradient (light-first)
- Headline alignment: Centered → left-aligned (Allura brand kit rule: headings always left-aligned)
- Accent underline: Red bar → Warm Yellow rounded bar (`80px × 4px`, `border-radius: 2px`)
- Mockup: Phone/device → Allura full mark or community imagery (see Section 4)
- Overall feel: Aggressive/luxury → warm/inviting

### 3.2 Card Styles

Amaysin uses sharp-edged cards with heavy single-layer shadows and sometimes red left borders. Allura replaces this with the 3-layer editorial shadow system.

**Standard Card:**
```css
.card {
  background: #FFFFFF;
  border-radius: 16px;
  padding: 2rem;
  box-shadow:
    0 4px 20px -4px rgba(20, 35, 41, 0.08),   /* Layer 1: Ambient */
    0 8px 16px -4px rgba(20, 35, 41, 0.12),   /* Layer 2: Directional */
    0 2px 4px 0 rgba(20, 35, 41, 0.16);       /* Layer 3: Contact */
  border: 1px solid rgba(20, 35, 41, 0.06);
  transition: box-shadow 0.3s ease, transform 0.3s ease;
}

.card:hover {
  box-shadow:
    0 8px 32px -4px rgba(20, 35, 41, 0.12),
    0 12px 24px -4px rgba(20, 35, 41, 0.16),
    0 4px 8px 0 rgba(20, 35, 41, 0.20);
  transform: translateY(-2px);
}
```

**Card with Accent Top Border (for archetype cards, proof points):**
```css
.card-accent-top {
  border-top: 4px solid #FFC300;
  border-radius: 0 0 16px 16px; /* top corners flat where border sits */
}
```

**Dark Mode Card:**
```css
.card-dark {
  background: #1E2E36;
  border: 1px solid rgba(245, 245, 245, 0.08);
  box-shadow:
    0 4px 20px -4px rgba(0, 0, 0, 0.30),
    0 8px 16px -4px rgba(0, 0, 0, 0.25),
    0 2px 4px 0 rgba(0, 0, 0, 0.40);
}
```

**Radius values:**
- Cards: `16px`
- Buttons: `12px`
- Small badges/pills: `9999px` (full round)
- Images inside cards: `12px` (1 step down from card)

### 3.3 Section Dividers

Amaysin uses bold red horizontal bars (`#E63946`, `4px` height, full-width or fixed-width) to separate sections. This creates aggressive visual breaks.

**Allura replacement — Warm Yellow accent rule:**

```css
.section-divider {
  width: 80px;
  height: 4px;
  background: #FFC300;
  border-radius: 2px;
  margin: 2rem 0;
}
```

**Usage rules:**
- Place below every H1/H2 heading, left-aligned with the text block
- `80px` width for standard headings, `120px` width for hero-level sections
- Never full-width — Allura dividers are punctuation, not fences
- Color: `#FFC300` (Warm Yellow) only. No blue or green dividers — keep the signal clear

**Alternative — Memory Keep Pattern Divider:**
For transitions between major sections (e.g., between Brand Overview and Color System), use a subtle pattern band:
```css
.pattern-divider {
  height: 120px;
  background-image: url('assets/patterns/memory-keep-pattern.svg');
  background-size: 200px;
  background-repeat: repeat-x;
  background-position: center;
  opacity: 0.08;
}
```
- Organic droplet motif at 8% opacity creates texture without distraction
- Use sparingly: once per deck, between slides 2 and 3

### 3.4 Page Navigation (1-2-3-4-5 Circles)

Amaysin uses bottom-centered numbered circles to indicate slide position. Allura retains the pattern but softens it.

```css
.slide-nav {
  display: flex;
  gap: 12px;
  justify-content: center;
  padding: 2rem 0;
}

.slide-nav-dot {
  width: 40px;
  height: 40px;
  border-radius: 50%;
  display: flex;
  align-items: center;
  justify-content: center;
  font: 500 14px/1 var(--font-body);
  color: #0581A7;
  background: transparent;
  border: 2px solid rgba(5, 129, 167, 0.3);

}

.slide-nav-dot.active {
  background: #FFC300;
  border-color: #FFC300;
  color: #142329;
  font-weight: 600;
}

.slide-nav-dot:hover:not(.active) {
  border-color: #0581A7;
```

**Changes from Amaysin:**
- Size: 48px → 40px (slightly more delicate)
- Active state: Red fill → Warm Yellow fill with Dark Gray text
- Inactive state: Gray fill → transparent with Deep Blue border
- Font: Bebas Neue → Inter Medium (numbers should be quiet, not shouting)
- Hover: Add subtle transition (Amaysin has none)

## 4. Photography / Mood Direction

### 4.1 Allura Photography Principles

From the Brand Kit (Section 6): Warm, authentic, community-focused. Golden hour warmth, soft natural light. Genuine human connection, diverse urban communities. Avoid stock photography, posed shots, cold clinical imagery.

### 4.2 "Brand in Action" Section (Slide 1 — Proof Points / Photo Gallery)

Amaysin's Slide 1 includes a photo gallery row showing the brand "in the wild." Allura replaces this with community-centered imagery.

**Image Treatment Pipeline:**
```css
.brand-photo {
  border-radius: 12px;
  object-fit: cover;
  width: 100%;
  height: 240px;
  /* Optional: subtle warm overlay for consistency */
  filter: saturate(0.95) contrast(1.02);
}

.brand-photo-wrapper {
  position: relative;
  border-radius: 12px;
  overflow: hidden;
}

.brand-photo-wrapper::after {
  content: '';
  position: absolute;
  inset: 0;
  background: linear-gradient(
    180deg,
    transparent 60%,
    rgba(20, 35, 41, 0.4) 100%
  );
  border-radius: 12px;
}
```

**Photo Selection Criteria:**
1. **Primary (Hero adjacent):** A diverse group of 3-5 people in an urban community setting — park, neighborhood market, community center. Warm afternoon light. Genuine interaction (laughter, conversation, collaboration).
2. **Secondary 1 (Digital context):** A person using a laptop or phone in a cozy cafe or home setting. Screen should show a warm interface (not cold blue light).
3. **Secondary 2 (Empowerment):** A community leader speaking, teaching, or organizing. Should feel grassroots, not corporate.
4. **Tertiary (Detail shot):** Hands — typing, holding a phone, sharing something. Close-up, shallow depth of field, warm tones.

**No-Go List:**
- Stock photography with "perfect" models
- Office environments with fluorescent lighting
- Isolated devices floating on white backgrounds
- Anything with a predominantly blue/white color cast

### 4.3 Logo Placement on Photos

When the Allura logo appears over photography (e.g., hero section alternative):
```css
.logo-over-photo {
  background: rgba(245, 245, 245, 0.92);
  padding: 1rem 1.5rem;
  border-radius: 12px;
  backdrop-filter: blur(8px);
}
```
- Never place the logo directly on a busy image without a scrim or card backing
- Minimum contrast ratio: 7:1 (WCAG AAA)

### 4.4 Mood Board Assets

The Allura mood board assets are referenced in `clients/allura-memory/assets/mood-board/` (or `assets/mood/`). If building the presentation locally, reference these paths:
- `assets/mood-board/community-gathering.jpg` — Primary hero vibe
- `assets/mood-board/digital-warmth.jpg` — Interface/context vibe
- `assets/mood-board/urban-connection.jpg` — City/community vibe

If mood board files are not yet present, use placeholder containers with the correct aspect ratios (16:9 for hero, 4:3 for gallery cards) and the `brand-photo` CSS class above.

## 5. Critical Visual Rules

### 5.1 What MUST Change from Amaysin Template

| Element | Amaysin | Allura |
|---------|---------|--------|
| **Color accents** | Red `#E63946` | Warm Yellow `#FFC300` |
| **Hero background** | Dark radial gradient | Warm light gradient |
| **Display typeface** | Bebas Neue (condensed, uppercase) | Outfit (geometric, sentence case) |
| **Body typeface** | Montserrat | Inter |
| **Card borders** | Red left border, sharp corners | Warm Yellow top accent, 16px radius |
| **Shadow style** | Heavy single-layer dark shadow | 3-layer editorial shadow system |
| **Section dividers** | Full-width red bars | 80px Warm Yellow rounded bars |
| **Nav dots** | Red fill, Bebas Neue numbers | Warm Yellow fill, Inter numbers |
| **Overall tone** | Aggressive, luxury, bold | Warm, inviting, community-centric |
| **Photography** | Product-focused, dramatic lighting | Community-focused, golden hour warmth |

### 5.2 What MUST Stay the Same for Quality Parity

| Element | Why It Stays |
|---------|--------------|
| **16:9 aspect ratio** | Standard presentation format — no change needed |
| **5-slide structure** | Proven narrative arc: Overview → Colors → Logo → Typography → Applications |
| **Grid-based layouts** | 12-column grid with 24px gutters is brand-agnostic and optimal |
| **Card-based information architecture** | Cards group related content visually — pattern is universal |
| **Numbered page navigation** | Accessibility and orientation aid; only styling changes |
| **Hero + supporting content rhythm** | Each slide leads with impact, then delivers detail — proven hierarchy |
| **High contrast text** | WCAG AAA is the target for both brands; contrast ratios must remain 7:1+ |
| **Generous whitespace** | Amaysin's padding (4-6rem sections) supports Allura's "breathing room" principle |

### 5.3 Accessibility Considerations

**WCAG AAA Compliance (Non-negotiable):**
- All text on `#F5F5F5`: use `#142329` (16.2:1) ✅
- All text on `#142329`: use `#F5F5F5` (16.2:1) ✅
- Large text on `#142329`: use `#FFC300` (11.8:1) ✅
- Large text on `#142329`: use `#BDBD0D` (7.2:1) ✅
- Body text on `#FFC300`: FAIL — never use Warm Yellow as a body text background
- Body text on `#0581A7`: FAIL (4.2:1) — only use Deep Blue for large text or UI elements, never body paragraphs

**Motion & Animation:**
- If adding transitions between slides: no parallax, no auto-play. Use simple opacity + translateY fades (`duration: 400ms`, `ease: cubic-bezier(0.4, 0, 0.2, 1)`).
- Respect `prefers-reduced-motion`: disable all transforms, use instant cuts.

**Focus States:**
```css
a:focus-visible,
button:focus-visible {
  outline: 3px solid #0581A7;
  outline-offset: 3px;
  border-radius: 4px;
}
```

**Image Accessibility:**
- All photographs: `alt` text required (min 10 words describing the scene and its emotional tone)
- Decorative patterns: `role="img"` + `aria-label="Decorative brand pattern"`
- Logo images: `alt="allura wordmark"` or `alt="allura icon"` depending on variant

**Minimum Touch Targets:**
- Navigation dots: 40px × 40px (already specified)
- Buttons: `min-height: 48px`, `min-width: 120px`, `padding: 12px 24px`
